OpenGame is an open-source agentic framework that lives in your terminal. Describe a game in plain language — a Snake clone, a tower defense, a side-scrolling platformer — and OpenGame plans the architecture, scaffolds the project, writes the code, runs it in a sandboxed browser, and iteratively debugs it until it is fully playable.

## How it works

OpenGame combines a purpose-built CLI with **Game Skill** — a two-part agent capability that separates stable scaffolding from systematic debugging.

<Steps>
  <Step title="Install OpenGame">
    Clone the repo, run `npm install && npm run build && npm link` to expose the `opengame` command on your PATH.
  </Step>
  <Step title="Set your API key">
    Export `OPENAI_API_KEY` (or swap in GameCoder-27B for maximum game-coding accuracy). Optionally configure image, video, and audio providers for full asset generation.
  </Step>
  <Step title="Describe your game">
    Create an empty folder, run `opengame`, and describe the game you want. OpenGame selects the right engine (canvas, Phaser, three.js), scaffolds the project, and writes the code.
  </Step>
  <Step title="Play and iterate">
    Open the generated `index.html` in your browser. Keep chatting with OpenGame to add features, fix bugs, or change the art style — it edits files and re-verifies playability after each change.
  </Step>
</Steps>
